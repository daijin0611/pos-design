# POS H5 高保真原型设计方案

## Context

POS 系统二期在现有 Java + Vue PC 管理系统基础上，新增微信公众号 H5 移动端。三端共用同一 H5 应用，通过角色路由区分：会员端、员工端、管理端。需求文档已完成，需要制作高保真 Figma 原型用于客户演示和开发参考。

## 视觉风格 — 健康美业

| 类别 | 值 |
|---|---|
| 主色调 | 薄荷绿 `#2CB5A0`（健康、专业） |
| 辅助色 | 玫瑰粉 `#F5A0B5`（温馨、美业） |
| 强调色 | 琥珀金 `#E8A849`（积分、优惠券） |
| 背景色 | `#F7F8FA` 浅灰白 |
| 文字主色 | `#333333` |
| 文字次色 | `#999999` |
| 卡片圆角 | 12px |
| 卡片阴影 | 0 2px 12px rgba(0,0,0,0.08) |
| 字体 | 系统默认（微信 H5 兼容） |

## Figma 文件结构

单文件 `POS-H5-Prototype.figma`，5 个 Page：

### Page 1: 设计系统
- 色彩变量、字体规范
- 公共组件：导航栏、标签栏、卡片、按钮、表单
- 图标库

### Page 2: 全局流程（4 页）
- 启动页 → 微信授权 → 角色判断
- 消息中心
- 设置页

### Page 3: 会员端（~20 页）
- 会员首页（余额、积分、优惠券入口）
- M1 消费余额查询（余额卡片 + 交易明细）
- M2 密码设置/修改
- M3 验证码接收
- M4 预约（选店→选项目→选时间→确认→我的预约）
- M5 优惠券（列表→详情→使用）
- M6 门店查询（地图+列表+详情）
- M7 会员二维码
- M8 积分（余额+明细）
- M9 积分兑换（商品列表→确认兑换）

### Page 4: 员工端（~15 页）
- 员工首页（今日提醒、快捷入口）
- E1 工资查询（月度总览+明细）
- E2 会员提醒-修复/调理（提醒列表+跟进）
- E3 会员提醒-生日（生日列表）
- E4 满意度分值（评分看板）
- E5 会员查询（搜索+列表+详情）
- E6 店长收银（选项目→购物车→结算→支付）

### Page 5: 管理端（~10 页）
- 管理首页（数据概览卡片）
- A1 业绩查询（门店/员工维度）
- A2 员工工资查询（总览+明细）
- A3 项目排名（排行榜）
- A4 员工业绩排名
- A5 员工工资排名

## 执行策略

分三批执行：
1. **第一批**：设计系统 + 全局流程 + 三端首页
2. **第二批**：会员端全部页面
3. **第三批**：员工端 + 管理端全部页面

## 技术约束

- 微信公众号 H5（非小程序）
- 技术栈：Vue3 + Vant 4
- 设计尺寸：375x812 (iPhone 标准)
- 顶部安全区 44px，底部安全区 34px
