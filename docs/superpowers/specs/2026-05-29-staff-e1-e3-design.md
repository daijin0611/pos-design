# POS H5 员工端 E1-E3 设计规格

> 基于 POS 项目二期需求文档，设计员工端一二级优先级功能（E1 工资查询、E2 回访提醒、E3 生日提醒）。
>
> 产出为 Vue3 + Vant 真实项目代码，使用 Mock 数据。

---

## 设计决策

| 决策项 | 选择 | 理由 |
|--------|------|------|
| 首页布局 | 方案 A 信息流式 | 按优先级排列卡片，员工一眼看到待办事项和关键数据，操作效率最高 |
| 导航模式 | 卡片式首页，无底部 Tab | 功能入口直接在首页卡片中展示，减少导航层级 |
| 视觉风格 | 保留薄荷绿+玫瑰粉+琥珀金 | 符合健康美业品牌调性，与一期设计系统一致 |
| 数据层 | Mock 数据 | 先用假数据完成前端，后续对接真实 API 时替换 |
| 技术栈 | Vue3 + Vant 4 + Vue Router + Pinia | 团队已有 Vue3 经验，Vant 适合移动端 H5 |

## 页面清单（共 7 页）

### 页面 1：员工首页（StaffHome）

**路由**：`/staff`

**信息流结构**（自上而下）：

1. **头部区域**
   - 薄荷绿渐变背景
   - 问候语「XX，早上好/下午好」+ emoji
   - 角色标签（高级技师）+ 所属门店
   - 保持轻量，不含统计数据

2. **今日待办摘要**
   - 暖色（琥珀浅底）卡片
   - 三个横排数据卡片：待回访数 / 本周生日数 / 今日已完成数
   - 每个数字可点击跳转对应页面
   - 用颜色区分：待回访红色、生日粉色、已完成绿色

3. **工资速览卡片**
   - 白色卡片
   - 大字显示本月总工资
   - 下方三行小字：劳动/产品/开卡分项
   - 右上角「查看详情 →」跳转 E1 工资页

4. **待跟进会员**
   - 合并展示回访提醒 + 生日提醒
   - 每条：头像 + 姓名 + 提醒类型 + 时间/状态
   - 颜色标签区分：超期红、待跟进绿、生日粉
   - 最多显示 3 条
   - 底部「查看全部 →」

5. **更多功能入口**
   - 3 列网格图标：工资查询 / 回访提醒 / 生日提醒
   - 后续扩展 E4-E6 时在此追加

**交互**：整个页面可滚动，无底部 Tab。

---

### 页面 2：工资总览（SalaryView）

**路由**：`/staff/salary`

**布局**：

1. **NavBar**：「工资查询」+ 返回箭头

2. **月份选择器**
   - 右对齐，左右箭头切换月份
   - 格式：「2026年5月」

3. **工资总额卡片**
   - 薄荷绿渐变大卡片
   - 显示：本月总工资（截至X日）
   - 大字 ¥8,640.00
   - 环比：较上月 ↑ ¥320（金色文字）

4. **分项卡片网格（2×2）**
   - 劳动工资：绿点 + 金额 + 「点击查看明细」
   - 产品提成：粉点 + 金额 + 「点击查看明细」
   - 开卡提成：金点 + 金额 + 「点击查看明细」
   - 其他：灰点 + 金额
   - 点击任一分项跳转 E1 明细页

---

### 页面 3：工资明细（SalaryDetail）

**路由**：`/staff/salary/:category`（category = labor | product | card | other）

**布局**：

1. **NavBar**：显示分类名（如「劳动工资明细」）+ 返回箭头

2. **总计**：该分类总金额

3. **明细列表**
   - 每条：项目名（如「肩颈调理（手工费）」）
   - 次行：日期 + 客户名（如「5月3日 · 李女士」）
   - 右侧：金额（绿色 +¥280）
   - 列表底部：「查看更多」加载

---

### 页面 4：回访提醒列表（ReminderList）

**路由**：`/staff/reminder`

**布局**：

1. **NavBar**：「回访提醒」+ 返回箭头

2. **Tab 筛选**
   - 三个胶囊标签：待回访(N) / 已完成 / 已超期(N)
   - 激活项用主色底 + 白字，非激活灰底
   - 已超期数量 > 0 时标签变红色底

3. **提醒卡片列表**
   - 每张卡片包含：
     - 左侧：圆形头像 + 会员 emoji
     - 中间：会员名 + 服务项目名
     - 下方：上次服务日期 + 回访周期说明
     - 超期项：红色边框 + 红色背景 + 「⚠ 已超期 N 天」
   - 底部双按钮：「已联系」（主色实心）+「📞 拨打」（描边）
   - 点击「已联系」弹出回访操作弹窗（页面 5）
   - 点击「拨打」调起系统电话（`tel:` 协议）

---

### 页面 5：回访操作（ReminderAction）

**形式**：Vant Popup 弹窗（从底部弹出），非独立路由页

**布局**：

1. **标题**：记录回访 — XX女士

2. **备注文本框**
   - 多行输入，placeholder「请输入回访备注...」
   - 浅灰背景

3. **快捷备注标签**
   - 一行可选标签：效果好 / 需继续 / 已预约下次 / 不满意
   - 可多选，选中变主色底
   - 点击标签自动填入文本框

4. **确认按钮**
   - 主色大按钮「确认提交」
   - 提交后：
     - 该提醒标记为已完成
     - 首页待办数 -1
     - Toast 提示「回访已记录」

---

### 页面 6：生日提醒（BirthdayList）

**路由**：`/staff/birthday`

**布局**：

1. **NavBar**：「🎂 生日提醒」+ 返回箭头

2. **统计**：右上角小字「本月共 N 位」

3. **本周生日（粉色底卡片组）**
   - 小标题：「📆 本周」+ 粉色文字
   - 每位会员：
     - 头像 + 姓名
     - 生日倒计时（明天 / 后天 / N天后）
     - VIP 等级 + 累计消费
     - 右侧「📞 致电」粉色按钮

4. **本月后续（灰色底卡片组）**
   - 小标题：「📆 本月后续」+ 灰色文字
   - 每位会员：
     - 头像 + 姓名
     - 生日日期 + 会员等级
     - 右侧「N天后」灰色文字
   - 点击卡片可查看会员详情

---

### 页面 7：会员档案速览（MemberProfile）

**路由**：`/staff/member/:id`

**用途**：公共页面，从回访提醒和生日提醒点击会员头像/姓名进入。

**布局**：

1. **NavBar**：会员姓名 + 返回箭头

2. **会员信息头部**
   - 头像 + 姓名 + 手机号
   - VIP 等级标签 + 会员卡号

3. **资产概览**
   - 三列：当前余额 / 累计消费 / 积分

4. **最近服务记录**（最多 5 条）
   - 每条：项目名 + 日期 + 服务员工
   - 底部「查看更多」

5. **底部操作栏**
   - 「📞 拨打电话」按钮

## 交互细节

### 回访操作流程

1. 员工在回访列表点击「已联系」→ 底部弹出 ReminderActionSheet
2. 可选填备注文本 + 点击快捷标签
3. 点击「确认提交」→ 更新提醒状态为 completed
4. Toast 提示「回访已记录」→ 弹窗关闭 → 列表刷新

### 电话拨打

- 点击「📞 拨打」或「📞 致电」→ `window.location.href = 'tel:138xxxx1234'`
- H5 在微信浏览器中会弹出拨号确认

### 月份切换

- 工资页面左右箭头切换月份
- Mock 数据提供近 6 个月数据
- 切换时卡片数字更新（Vant Skeleton 骨架屏加载态）

### 空状态

- 回访列表无数据时：插画 + 「暂无待回访会员，休息一下吧 🎉」
- 生日列表无数据时：插画 + 「本月暂无会员过生日」
