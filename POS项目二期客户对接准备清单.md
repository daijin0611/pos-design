## 四、建议的对接议程

### 第一次对接（需求确认会，约 2 小时）

| 时长  | 环节         | 内容                                                         |
| ----- | ------------ | ------------------------------------------------------------ |
| 15min | 一期回顾     | 快速演示一期现有功能，对齐双方认知                           |
| 30min | 技术方案讲解 | 公众号 H5 方案说明，对比小程序的优劣，展示参考案例           |
| 45min | 需求逐条确认 | 按四端逐条过需求，重点确认上方「业务规则确认」中的 10 个问题 |
| 15min | 资质准备说明 | 告知客户需要准备的材料清单，明确交付时间                     |
| 15min | 排期与下一步 | 确认优先级，沟通大致时间节点，约定下次对接时间               |

### 会后跟进

- [ ] 整理会议纪要，邮件发送客户确认
- [ ] 跟进资质材料准备进度（公众号服务号优先）
- [ ] 开始搭建 H5 项目骨架（不需要等资质，本地开发先行）
- [ ] 确认客户方对接人（日常沟通、需求答疑、验收测试）

## 五、需要注意的风险预沟通

| 风险       | 沟通要点                                                       |
| ---------- | -------------------------------------------------------------- |
| 公众号审核 | 告知客户服务号认证是第一优先级，审核通过前无法进行微信授权联调 |
| 需求变更   | 约定需求冻结节点：第一阶段开发启动后不再新增需求变更           |
| 数据质量   | 提前沟通会员手机号/生日数据不完整的情况，可能需要线下补录      |
| 第三方对接 | 订货宝和银行对接工作量有不确定性，需单独评估                   |
| 验收标准   | 约定每阶段交付后的验收周期（建议 3-5 个工作日）                |
